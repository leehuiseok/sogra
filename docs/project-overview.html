프로젝트 개요

Sogra v1

음식점 사장님이 날씨, 요일, 시즌 같은 상황 신호를 바탕으로 포스터, 릴스, SNS 캡션을 생성하고 최종 산출물을 다운로드하거나 복사한 뒤 본인 SNS 계정에 직접 업로드하도록 설계된 AI 마케팅 자동화 SaaS입니다.

Next.js 14 App Router Supabase Auth, DB, RLS, Storage Inngest Background Jobs AI Content Generation Manual SNS Upload Toss Payments

요약

누구를 위한 서비스인가

카페, 일반 음식점, 배달 음식점 사장님을 대상으로 합니다. 마케팅 문구 작성, 이미지 제작, 릴스 제작, 게시 타이밍 판단을 한 화면에서 처리하는 것이 목표입니다.

무엇을 자동화하는가

상황 추천, 자유 입력 NLU 분석, 콘텐츠 브리프 구성, 캡션과 포스터 생성, 릴스 비동기 생성, 최종 산출물 전달을 자동화합니다.

핵심 제약

v1의 인스타그램 인증과 자동 게시 흐름은 제외되어 있습니다. 사용자는 포스터와 릴스를 다운로드하고 캡션을 복사해 직접 업로드합니다.

핵심 사용자 흐름

1
회원가입과 로그인

Supabase Auth를 사용합니다. 인증된 사용자는 앱 레이아웃에서 온보딩 완료 여부를 검사합니다.

2
온보딩 1~4단계

업종 선택, 국외이전 동의, 매장명과 주소, 대표 메뉴, 브랜드 톤 키워드를 store_profiles에 저장합니다.

3
상황 추천 또는 자유 입력

추천은 날씨와 달력 신호를 점수화합니다. 자유 입력은 Claude 기반 NLU로 event, action, when, target을 추출하고 신뢰도에 따라 확인 UI를 띄웁니다.

4
상황 트리거 저장

추천 또는 자유 입력 결과는 situation_triggers에 저장됩니다. NLU 이력은 nlu_parse_events에 남겨 추후 보정과 드리프트 분석에 사용합니다.

5
콘텐츠 생성

/api/content/generate가 캡션, 포스터, 릴스 브리프를 만들고 월간 할당량을 차감합니다. 캡션은 텍스트 모델, 포스터는 이미지 모델, 릴스는 Inngest 잡으로 처리합니다.

6
미리보기, 승인, 산출물 전달

생성된 콘텐츠는 marketing_contents에서 상태를 관리합니다. 승인된 포스터와 릴스는 다운로드하고, 캡션은 복사해 사용자가 직접 업로드합니다.

7
운영 관리

할당량, 생성 실패 환불, signed URL 갱신, 결제 상태, 외부 API 비용을 추적합니다. Instagram 게시와 성과 측정은 v1 필수 흐름에서 제외되었습니다.

외부 연동

AI Providers

  • 캡션: Claude Haiku 기본, GPT-4o-mini fallback 구조
  • 포스터: DALL·E 3 이미지 생성 후 Supabase Storage로 복제
  • 릴스: Runway Gen-3 비동기 생성, Pika fallback 인터페이스 보유
  • MOCK_AI_PROVIDERS=true면 외부 API 없이 결정적 mock 결과 반환

SNS 수동 업로드

  • 온보딩에서 Instagram OAuth를 요구하지 않습니다.
  • 포스터와 릴스는 다운로드 가능한 최종 산출물로 제공합니다.
  • 캡션은 복사 버튼으로 제공합니다.
  • 사용자가 본인 Instagram 앱 또는 웹에서 직접 업로드합니다.

Toss Payments

  • 월 49,000원 단일 플랜 구조가 DB에 정의되어 있습니다.
  • 웹훅은 서명 검증과 customer key 매핑을 전제로 저장됩니다.
  • MOCK_TOSS=true로 베타 환경을 지원합니다.

Signals

  • 날씨: OpenWeather 기반 신호
  • 달력: 요일과 공휴일 근접도

운영 체크포인트

  • Instagram OAuth와 자동 게시 재도입 전에는 App Review 권한, 토큰 저장, 실패 복구, 사용자 동의 UX를 별도 검증해야 합니다.
  • AI real 호출 전 MOCK_AI_PROVIDERS=false와 각 API 키, 비용 모니터링을 확인해야 합니다.
  • Supabase Storage는 private bucket이므로 signed URL 만료와 assetRehostRefresh 동작을 확인해야 합니다.
  • 월간 quota는 DB RPC에서 차감되므로 실패 시 dead-letter 환불 흐름까지 E2E로 확인해야 합니다.
  • 결제 정식 활성화 전 Toss 웹훅 서명 검증, customer key 매핑, dunning grace period를 실환경에서 재검증해야 합니다.
  • NLU 모델명, 가격 추정치, OpenAI/Anthropic/Runway API 사양은 출시 시점에 다시 확인하는 것이 좋습니다.

이 문서는 현재 저장소의 코드, 마이그레이션, README 기준으로 작성되었습니다. 주요 진입점은 README.md, app/, lib/, inngest/, supabase/migrations/입니다.
